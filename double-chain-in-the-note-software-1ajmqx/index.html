笔记软件中的「双链」

发表于 2024-07-27

作为一定程度上社区驱动的产品，对产品来说，有品位的用户的限制是满足自己需求后失去反馈的动力；有能力的用户的限制是不满足自己需求后新启灶台。

思源笔记​走到今日离不开许多目光如炬的大佬们有理有据积极的反馈，其中不少真知灼见沉淀在论坛的回复和 issue 里。是难得的，对理解双链、理解笔记、理解一个产品为何是这样有帮助的材料。

分享一些与双链相关的讨论。

首先是关于「文档粒度」和「反链转移」的两个 issue、以及关于双链与标签的评论。 这几篇讨论在理念上似乎遥遥领先于竞品。

现在一些笔记软件都开始引入数据库，似乎缺少这样的哲学 🧐。

  • 文档粒度：打通正文与文件树的壁障 · Issue #556 · siyuan-note/siyuan · GitHub
  • 反链转移：支持转移引用 · Issue #4025 · siyuan-note/siyuan · GitHub
  • 双链与标签：实践中对双链和写作的一些思考 - 链滴

链接右键菜单「转换为」的设计：

  • 探讨：划选而后新建被引用之文档，是否应默认设置静态锚文本 - 链滴
