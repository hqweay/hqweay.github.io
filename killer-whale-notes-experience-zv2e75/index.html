虎鲸笔记体验

发表于 2025-12-20

中度体验了一两周虎鲸笔记，尝试了软件内各种功能，并通过基于（利用）软件本体的设计以及开发插件的方式实现一些工作流，最后决定暂且搁置将其放在观望状态。

比起目前使用的思源笔记，虎鲸笔记对我来说，优势在于设计的统一，以及基于此双链操作的一致性，比方说：

  • 块的粒度统一：页面、块、标签 可以当作一回事

  • 可以自底向上构建视图：卡片、看板、日历、表格……

    • 在任意位置记录数据并打上标签，可以通过查询块汇总信息并以某种视图展示
      ​

  • 镜像块：思源目前没有镜像块。但这点其实是双链笔记的基操。

  • 对「反链面板」的改进：虎鲸笔记的「反链面板」支持分 tab 展示，并且可以保存自定义查询。这样它的功能就不仅是「反链」的面板，同时也是任何汇总信息的面板。【这点与底层架构关系不大，思源倒是可以通过插件实现】

不过我感觉虎鲸笔记对标的是 Logseq，然后参考 Tana 引入了一些新特性。如果之前是 Logseq 的用户，可能解决了不少痛点并提供了不少新功能。（不过 Logseq 的 DB 版本发布后可能差距就不大了）

这个功能和目前虎鲸笔记中的「内联页面」「转化为-镜像」的作用很相似：

由此我想，作者在开发时可能没有参考过类似应用，甚至双链开创者的设计——这是最好的情况。如果作者参考了，但是觉得这个功能不重要，那可能没有很深入地使用双链的一些场景。

不过这也不是什么问题，我提了个 issue，作者也表示有机会会改进。

除了双链的基本操作，作者对双链的改进也避免尝试。比方说关于链接与引用的转换，作者以「这违反了双链笔记的初衷」回复。

这个说法就稍微有点原教旨了……但也不是什么问题。

从软件的设计就能看出，作者本身确实是一个资深、至少有深度使用双链的用户。开发者有自己的品味是好事，我使用了太多国内的产品，有品味的开发者太少了。这能保证应用的下限——至少都能满足和作者想法类似的用户。

但作者对双链的应用同时缺少一些「基础用法」以及「改进用法」，我想它短时间可能不会有太高的上限。

我也可以尝试说服，争取作者开发方向上的调整，但是：

一方面，软件的设计有作者自己的品味与考量。我的想法可能有局限。比方说也许我提到的工作流，可以使用目前软件的设计用其它方式实现。如果开发者有足够的品味，其实忘记自己的工作流，去体会软件的设计，按照开发者的思路去使用应用是最合适的。不过我尝试了下，目前还是有点别扭。

另一方面，这种反馈太费劲了。而且虎鲸笔记是一款闭源、收费的应用，且由一人维护。自双链流行以来，我也尝试、关注过不少应用，很多都昙花一现。RoamEdit、葫芦笔记……虎鲸笔记的宣传视频下有人评论「我不愿再陪哪个笔记成长了，太累了。」

投入一款应用也得考虑沉没成本……

不过这些都是基于个人体验的「增量感受」，对不同人来说感受肯定不一样。任何产品还是得自己体验下，只有合不合适，没有好不好。对我来说，「虎鲸笔记」的好还是很显眼了，不然我也不会关注。

最后列一下对我来说，目前缺少的点吧。如果设计上能满足，其它其实都可以自己写插件实现……不过这种投入是否值得还得观望观望。

  • 缺少与服务端通信的接口：目前可以包装 mcp API 使用，问题不大
  • 剪藏网页：目前 mcp 提供的 API 可以实现
  • 收藏网站聚合数据库：目前还需要一个插入标签的 API
  • 图片图床：目前插件 API 可实现
  • 发布博客：目前插件 API 可实现
  • 笔记导入：目前可自己定制化利用插件 API 实现
  • 笔记导出：目前可自己定制化利用插件 API 实现
  • 格式化文本：目前有插件，也可自己再实现
  • 移动端
  • 同步 voicenotesai 的数据：目前插件 API 可实现
  • 标签的属性不支持公式：问题不大？
  • 标签的属性不支持默认值
  • 不能达到随机 SQL 的效果：给开发者提了，说排序会增加一个随机字段。【 v1.56.0 更新：查询增加随机排序 】
  • replace with 的设计：给开发者提了，可能会改进【 v1.56.0 更新：「内联页面」功能改为「内联/交换页面」，如果被内联的块不是顶级块的话会在原地留一个引用，也就是交换行为 】
  • 属性的网络图片目前不能一键转本地
  • 没有找到方法查询所有的二级标题块
  • 组合使用块引时，反链列表不能方便地排除某些块引
